# 回测系统数据加载问题分析

## 📋 问题概述

quant_trade-main 项目的回测系统存在严重的数据加载问题，导致无法在Mac/Linux系统上运行。

## 🔍 具体问题

### 1. 硬编码Windows路径 (main.py)

**问题代码**:
```python
def load_all_stock_data():
    """直接遍历指定路径下的所有CSV文件并合并"""
    # 固定路径 - 硬编码Windows路径
    target_dir = r"E:\stock\backtest\data\analyzed\5min"
```

**影响**:
- 只能在Windows系统特定目录下运行
- 无法在Mac/Linux上运行
- 缺乏可移植性

### 2. 不一致的数据加载逻辑

**两个不同的数据加载实现**:

1. **main.py**中的`load_all_stock_data()`:
   - 硬编码路径
   - 批量加载所有CSV文件
   - 文件名解析逻辑简单

2. **data_loader.py**中的`load_data()`:
   - 使用相对路径 `"../../data/analyzed/{timeframe}"`
   - 单个股票加载
   - 更严格的验证

**矛盾**:
- `main.py`不使用`data_loader.py`
- 两套不同的路径假设
- 数据格式期望不一致

### 3. 路径不存在问题

**实际目录结构**:
```
quant_trade-main/
├── backtest/
│   ├── main.py          # 期望 E:\stock\backtest\data\analyzed\5min
│   ├── src/utils/data_loader.py  # 期望 ../../data/analyzed/
│   └── [没有data目录]    # 实际不存在
```

**问题**:
- 两个数据加载器期望的路径都不存在
- 项目缺少实际数据目录
- 没有示例数据或配置说明

### 4. 平台依赖问题

**Windows特定代码**:
- `r"E:\stock\backtest\data\analyzed\5min"` (原始字符串，Windows路径)
- 硬编码反斜杠路径分隔符
- 假设Windows文件系统结构

### 5. 缺乏配置和灵活性

**缺失功能**:
- 没有配置文件支持
- 不能通过环境变量配置路径
- 不支持命令行参数指定数据目录
- 缺乏数据源抽象（仅支持本地CSV）

## 📊 数据格式分析

### 期望的数据格式

从代码分析，系统期望CSV文件包含以下列:

**必需列**:
- `trade_date`: 日期时间列（解析为DatetimeIndex）
- `open`: 开盘价
- `high`: 最高价  
- `low`: 最低价
- `close`: 收盘价

**可选/附加列**:
- `symbol`: 股票代码（某些代码会添加此列）
- 其他技术指标列

### 文件命名约定

从代码推断:
- 文件名包含股票代码，如 `000063.SZ.csv` 或 `000063.SZ_analysis.csv`
- `main.py`会去掉 `_analysis` 后缀和 `.csv` 扩展名作为symbol

### 目录结构期望

**main.py 期望**:
```
E:\stock\backtest\data\analyzed\5min\
  000001.SZ.csv
  000002.SZ.csv
  ...
```

**data_loader.py 期望**:
```
backtest/data/analyzed/5min/
  000001.SZ.csv
  000002.SZ.csv
  ...
```
