## 1. Advanced RSI Momentum with Trend Dot [JoeyWave]

- **URL**: https://www.tradingview.com/script/vj2ej6ux-Advanced-RSI-Momentum-with-Trend-Dot-JoeyWave/
- **Core Logic**: RSI(14) 上叠加 EMA(9) 和 WMA(45) 两条均线,通过快慢均线关系判断趋势方向和动量状态,配合动量区间色带和趋势指示点.
- **Technical Indicators**: RSI(14), EMA(9) of RSI, WMA(45) of RSI
- **Strategy Type**: 动量
- **Market/Timeframe**: 任意市场/任意时间框架
- **Innovation**: 三层分析合一: RSI 本身(超买超卖) + EMA/WMA 交叉(趋势方向) + 动量区间色带(75-90 强多头, 10-25 强空头). 趋势点(Trend Dot)可跟随 RSI 线或固定在角落, 5种形状可选, 三角形/箭头会随趋势方向自动翻转.
- **借鉴**: RSI 上叠加双均线的思路, 把传统 RSI 从超买超卖工具升级为趋势+动量综合判断工具. WMA 相比 SMA 对近期数据更敏感.

## 2. Aegis HMA Sentinel [wjdtks255]

- **URL**: https://www.tradingview.com/script/WR4lI96h/
- **Core Logic**: 基于Hull MA的多层过滤趋势追踪系统,结合高时间框架过滤、成交量确认(1.5x均值检测机构参与)、K线动量验证,只在高信念突破时入场.
- **Technical Indicators**: HMA (Hull Moving Average), HTF Filter (高时间框架), Volume Filter, Candle Momentum
- **Strategy Type**: 趋势跟踪
- **Market/Timeframe**: 通用
- **Innovation**: "White Line Guardian"概念: 一条长期基线作为结构过滤器,阻止在熊市周期做多、牛市周期做空. 内置实时分析面板显示胜率、累计PnL、R/R、推荐仓位大小.
- **借鉴**: 多维过滤引擎(Multi-Layer Filtering Engine)的架构设计 -- 趋势同步 + 长期基线守护 + 成交量过滤 + K线动量过滤, 只有全部通过才发出信号.

## 3. Multi-Timeframe Confluence System [forexobroker]

- **URL**: https://www.tradingview.com/script/87TCuLDg-Multi-Timeframe-Confluence-System-forexobroker/
- **Core Logic**: 在4个可配置高时间框架+当前图表上,分别分析趋势方向(三EMA系统)、动量状态(RSI+MACD)、市场结构(HH/HL/LH/LL),当足够多维度跨时间框架一致时才发出信号.
- **Technical Indicators**: Triple EMA(9/21/50), RSI(14), MACD Histogram, Swing Pivot Analysis
- **Strategy Type**: 趋势跟踪(多时间框架共振)
- **Market/Timeframe**: 通用, 推荐 15m/1H/4H/Daily
- **Innovation**: 三维分析(Trend+Momentum+Structure) x 五个时间框架的矩阵评分系统. Quality Score 1-10 量化信号强度, 可用于仓位管理(Q:8+全仓, Q:6-7半仓, <Q:6跳过). 信号冷却期防止聚集. 仅4个request.security()调用,高效.
- **借鉴**: 多时间框架共振是提升胜率的核心方法. Confluence Score 的设计可以系统化地量化"多个条件一致"的程度, 用于动态调整仓位.

## 4. AG Pro Daily Open Acceptance Map [AGPro Series]

- **URL**: https://www.tradingview.com/script/yYmrm0cK-AG-Pro-Daily-Open-Acceptance-Map-AGPro-Series/
- **Core Logic**: 以当日开盘价为核心锚点,追踪价格在开盘价上方的接受(Bullish Acceptance)、下方的接受(Bearish Acceptance)或反复失败(Testing),构建状态框架帮助日内决策.
- **Technical Indicators**: Daily Open Level, Acceptance State Model
- **Strategy Type**: 支撑阻力(日内)
- **Market/Timeframe**: 日内交易, 任意时间框架
- **Innovation**: 不是简单地画一条开盘价线,而是构建了完整的状态机: Bullish Acceptance / Bearish Acceptance / Testing / First Reclaim. 把开盘价从被动参考线升级为主动决策框架. 分离当前/前一日开盘价形成视觉层级.
- **借鉴**: Daily Open 是机构交易者最常用的日内锚点之一. Acceptance/Rejection 状态机的思路可以应用到其他关键价位.

## 5. Chandelier Exit PRO (Fixed)

- **URL**: https://www.tradingview.com/script/VPqgZNz9-Chandelier-Exit-PRO-Fixed/
- **Core Logic**: 基于ATR构建动态止损位,结合ADX过滤横盘市场,在趋势转折时产生买卖信号.
- **Technical Indicators**: ATR (动态止损), ADX (趋势强度过滤)
- **Strategy Type**: 趋势跟踪(止损/出场)
- **Market/Timeframe**: 通用
- **Innovation**: 经典 Chandelier Exit 的增强版, 加入 ADX 过滤避免在横盘市场产生假信号. ATR 倍数止损可以根据波动性自适应调整止损距离.
- **借鉴**: ATR-based 动态止损是趋势跟踪系统的标准做法, ADX 过滤是提升信号质量的经典手段.

## 6. Volatility Regime Classifier [JOAT]

- **URL**: https://www.tradingview.com/script/dAC7QrzD-Volatility-Regime-Classifier-JOAT/
- **Core Logic**: 通过三个独立指标(ATR Z-score, ATR百分位, EMA方向比率)将市场分类为四种波动率状态: TRENDING/RANGING/VOLATILE/MIXED,并根据状态自适应调整ATR包络带宽度.
- **Technical Indicators**: ATR(14), ATR Z-Score, ATR Percentile, EMA Directional Ratio (Fast 20 / Slow 50)
- **Strategy Type**: 波动率(市场状态分类)
- **Market/Timeframe**: 通用
- **Innovation**: 不回答"市场有多波动"而是回答"市场处于什么类型的波动环境". 三个指标的优先级逻辑: VOLATILE(Z-score>2) > RANGING(percentile<35%) > TRENDING(EMA分离>1.5 ATR) > MIXED(兜底). 每种状态对应不同的ATR包络带乘数(3.0/2.0/1.5/1.0). RGB通道EMA平滑颜色过渡.
- **借鉴**: 市场状态分类是量化策略的核心前置步骤. 四状态模型(Trending/Ranging/Volatile/Mixed)映射到四种策略姿态(趋势跟踪/均值回归/减仓/观望)的思路非常实用. ATR Z-score 用统计方法检测异常波动, ATR percentile 自适应于品种特性.

## 7. TAO Dux Scalper v4.1 [5M]

- **URL**: https://www.tradingview.com/script/mNIAfKUD/
- **Core Logic**: 5分钟多条件共振剥头皮系统,综合成交量(RVOL/Z-score/Clímax)、市场结构(HH/HL/Sweep)、趋势(EMA/VWAP/HTF)和会话过滤,通过0-8分评分系统只执行高概率交易.
- **Technical Indicators**: EMA(9/21/50), VWAP, RVOL, Volume Z-score, ATR, BBWP, Bollinger Bands Width Percentile
- **Strategy Type**: 剥头皮/动量
- **Market/Timeframe**: 5分钟, 适用于期货/外汇
- **Innovation**: 完整的评分系统(0-8分): 体积(RVOL) + HTF趋势 + 结构 + Kill Zone + 流动性 + 波动率. 只有总分>=阈值才执行. 多种入场触发: 突破/回踩/EMA pullback/VWAP reclaim/恐慌/Clímax. 智能退出: 成交量崩溃/趋势翻转/超时. 内置Dashboard显示实时胜率/PF/回撤.
- **借鉴**: 评分系统比单一信号更稳健. 把"成交量是核心验证"的理念系统化 -- 价格没有成交量配合不可信. Kill Zone 会话过滤避免低流动性时段.

## 8. Panic Oversold & Volume Capitulation v3

- **URL**: https://www.tradingview.com/script/do7vIwle-Panic-Oversold-Volume-Capitulation-v3-With-Dashboard/
- **Core Logic**: 检测极端超卖+成交量投降的多条件组合: RSI(14)<30, RSI(6)<25, ATH回撤>-30%, Stoch K<15, 成交量>中位数, 成交量比>1.2, VIX>25.
- **Technical Indicators**: RSI(14), RSI(6), Stochastic K, Volume, Volume Ratio, VIX
- **Strategy Type**: 均值回归(极端超卖反转)
- **Market/Timeframe**: 股票市场, 日线级别
- **Innovation**: 多维度确认极端超卖: 双RSI(长期+短期) + 随机指标 + ATH回撤幅度 + 成交量放量 + VIX恐慌. 只有在极度恐慌时才考虑抄底.
- **借鉴**: 底部确认需要多个维度共振. VIX>25 作为宏观恐慌过滤, ATH回撤>-30%确保跌幅足够大, 成交量>中位数+成交量比>1.2确认恐慌性抛售.

## 9. SMT Divergence | Multi-Index Liquidity Comparison

- **URL**: https://www.tradingview.com/script/TGNPm7td-SMT-Divergence-Multi-Index-Liquidity-Comparison/
- **Core Logic**: 追踪相关指数期货(NQ/ES/YM)之间的结构性背离,当一个市场创新高/低而另一个未确认时标记背离,可视化跨市场分歧.
- **Technical Indicators**: Swing High/Low Comparison across correlated instruments
- **Strategy Type**: 订单流(跨市场结构背离)
- **Market/Timeframe**: 指数期货(NQ/ES/YM), 多时间框架
- **Innovation**: SMT(Smart Money Technique)背离的概念: 相关市场应同步运动, 当一个市场创新高而另一个未确认,暗示机构资金的流动性陷阱/转移. 将多品种对比、会话参考、高时间框架结构整合为一个工具.
- **借鉴**: 跨市场确认是机构交易的核心概念. 相关品种的背离往往预示着流动性事件或趋势反转. 可以扩展到其他相关品种对(如黄金/白银, 澳元/纽元等).

## 10. MTF DEMA Trend System + SuperTrend

- **URL**: https://www.tradingview.com/script/5csUbQ9q/
- **Core Logic**: 基于四个双指数移动平均线(DEMA)的多时间框架趋势系统,结合SuperTrend过滤器,当所有条件对齐时产生高概率信号.
- **Technical Indicators**: DEMA (Double EMA), SuperTrend
- **Strategy Type**: 趋势跟踪
- **Market/Timeframe**: 趋势市场, 多时间框架
- **Innovation**: DEMA 相比 EMA 减少了滞后性同时保持平滑性. 四个独立MTF DEMA + SuperTrend 确认的多层趋势验证. 平滑可视化减少高时间框架的阶梯失真.
- **借鉴**: DEMA 是减少均线滞后性的好方法, 计算公式: DEMA = 2*EMA - EMA(EMA). 多时间框架DEMA共振可以更准确地确认趋势.

## 11. Quantum Liquidity Map - VP, VWAP & CVD Confluence [NikaQuant]

- **URL**: https://www.tradingview.com/script/5xom3FAB-Quantum-Liquidity-Map-VP-VWAP-CVD-Confluence-NikaQuant/
- **Core Logic**: 将三个机构订单流方法整合为一个工具: 可见范围成交量剖面(VP)找到POC/VAH/VAL, 锚定VWAP+SD带测量价格偏离公允值的程度, CVD背离检测动量是否真实, 三者共振时才是高概率设置.
- **Technical Indicators**: Volume Profile (POC/VAH/VAL), Anchored VWAP with SD Bands, CVD (Cumulative Volume Delta) with Divergence
- **Strategy Type**: 订单流
- **Market/Timeframe**: 5分钟-4小时(日内), 1小时-日线(波段); 适用于流动性期货/主流外汇/大盘股/加密货币永续合约
- **Innovation**: 三种工具各有盲区,组合后互补: VP定位关键价格但没有方向, VWAP测量偏离但没有结构, CVD检测真假但缺少价位. 最佳设置: VAH边缘+VWAP外2SD+CVD背离三者同时出现. CVD用close-location-within-range方法估算(非tick级). Naked POC(未回测的上一周期POC)作为磁力位. ATR门控(1.5x ATR)过滤CVD背离噪音.
- **借鉴**: 多工具共振的核心思想: 任何单一指标都有盲区, 组合使用可以大幅提升信号质量. CVD背离的ATR门控思路(只有波动>1.5x ATR的摆动才算有效背离)是过滤噪音的好方法. VWAP锚定周期的选择应匹配交易周期.

## 12. LinReg Scale-In Strategy

- **URL**: https://www.tradingview.com/script/dpkq3n4O/
- **Core Logic**: 使用线性回归相关系数(Pearson's r, -1到+1)衡量趋势强度,随趋势增强分步建仓(在0.25/0.50/0.75三个阈值加仓),趋势衰弱时(r<0.25)一次性全部平仓.
- **Technical Indicators**: Linear Regression Correlation Coefficient (Pearson's r)
- **Strategy Type**: 趋势跟踪(分步建仓)
- **Market/Timeframe**: 商品/日线
- **Innovation**: 用统计相关系数而非价格/均线来判断趋势强度, 更加量化. 分步建仓(Scale-In)策略: 趋势越强仓位越大. "让利润奔跑, 快速止损"的哲学: 平仓条件只有一个(r<0.25), 比入场条件宽松得多. 最多金字塔3次.
- **借鉴**: Pearson's r 作为趋势强度指标比传统ADX更直观(-1到+1的范围). 分步建仓的核心: 趋势确认后逐步加仓, 趋势结束一次性退出. 简单但优雅的策略设计.

## Key Takeaways / Patterns Across Pages 22-28

### 1. 多时间框架共振是核心主题
- MTF Confluence System, MTF DEMA + SuperTrend, Aegis HMA Sentinel 都采用了多时间框架分析
- 核心: 当前时间框架的信号必须在更高时间框架方向上才能执行

### 2. 评分系统替代单一信号
- TAO Dux Scalper (0-8分), MTF Confluence (Quality Score 1-10) 都采用评分制
- 好处: 可以根据分数动态调整仓位大小, 而非简单的"做/不做"

### 3. 波动率状态分类越来越流行
- Volatility Regime Classifier 的四状态模型(Trending/Ranging/Volatile/Mixed)很有实用价值
- 可以作为策略的前置过滤器: 不同状态用不同策略

### 4. 订单流工具组合使用
- Quantum Liquidity Map 把 VP + VWAP + CVD 三合一
- 核心洞察: 任何单一工具都有盲区, 组合使用大幅提升信号质量

### 5. 日内锚点的创新使用
- AG Pro Daily Open Acceptance Map 把开盘价从简单线条升级为状态机
- 状态机思维(Bullish Acceptance / Bearish Acceptance / Testing / Reclaim)可以应用到其他关键价位

### 6. 成交量是核心验证层
- 几乎所有高质量策略都包含成交量过滤
- RVOL, Volume Z-score, 成交量比值是三种最常用的成交量分析方法

### 7. 统计方法替代经验判断
- LinReg Scale-In 用 Pearson's r 替代均线判断趋势
- Volatility Regime Classifier 用 Z-score 替代固定阈值
- 量化交易的趋势: 用统计学工具替代交易员的经验判断
