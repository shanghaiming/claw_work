## 1. Persistency with Moving Averages

- **策略名称**: Persistency with Moving Averages
- **URL**: tradingview.com/scripts/ (by finallynitin)
- **核心逻辑**: 追踪价格在关键移动平均线上方或下方持续停留的bar数量，衡量趋势"持久性"（Persistency）。持续在均线同一侧的bar越多，趋势越强，延续概率越高。
- **技术指标**: EMA/SMA (最多4条)，可自定义长度(如10/21/50/200)
- **策略类型**: 趋势跟踪 / 持续性分析
- **适用市场/时间框架**: 日线及以上时间框架效果最佳（最大持久性统计仅适用于日线+）
- **创新点**:
  - 引入Pradeep Bonde的"持久性"概念：不仅看趋势方向，更看趋势韧性
  - "决定性退出"(Decisive Exit)逻辑：当价格显示走弱迹象时启动待定退出，跌破退出水平时确认退出
  - 最大持久性(Max Persistency)统计：追踪价格在均线上方/下方停留的最长连续bar数
  - 方向性头部指示器(↑/↓/→)：汇总所有均线的计数方向

## 2. Trendlines with Breaks [EXCAVO]

- **策略名称**: Trendlines with Breaks
- **URL**: tradingview.com/scripts/ (by EXCAVO)
- **核心逻辑**: 通过连接连续的摆动高点和摆动低点绘制趋势线，当新pivot与活跃趋势线对齐(ATR容差内)时延伸而非创建重复线，自动检测突破并移除被突破的趋势线。
- **技术指标**: Swing Pivots (ta.pivothigh/ta.pivotlow)，ATR (用于合并容差)
- **策略类型**: 趋势线 / 支撑阻力 / 突破检测
- **适用市场/时间框架**: 通用
- **创新点**:
  - 合并而非重复：当连续pivot与活跃趋势线对齐时延伸线段而非创建新线
  - 历史验证：仅保留3个及以上pivot触碰且无pivot穿越到错误侧的趋势线
  - 缩放安全扩展：使用价格钳制而非extend.right，避免Y轴被极端投影压缩
  - 滚动缓冲区存储最多50个pivot用于趋势线构建和验证

## 3. Merged EMA Crossover & Pullback Analyzer

- **策略名称**: Merged EMA Crossover & Pullback Analyzer
- **URL**: tradingview.com/scripts/ (by orbsaleus)
- **核心逻辑**: 结合EMA交叉趋势跟踪与波动率回撤分析。快慢EMA交叉产生信号(遵循Ed Seykota原则：慢线周期至少是快线的3倍)，同时动态识别价格波段并标注最大回撤幅度。
- **技术指标**: EMA交叉(可配置快慢周期)，波段分析器(MoveFinder)
- **策略类型**: 趋势跟踪 + 波动率分析
- **适用市场/时间框架**: 通用，适合趋势明确的市场
- **创新点**:
  - 双重分析合一：EMA交叉趋势方向 + 波段回撤幅度分析
  - 动态标签显示：标注波段大小(百分比/点/点数)和具体回撤幅度
  - 帮助交易者根据历史回撤行为校准移动止损距离
  - 使用User-Defined Types (UDT)管理复杂的线条和标签绘制

## 4. Volumetric Structure Engine [JOAT]

- **策略名称**: Volumetric Structure Engine
- **URL**: tradingview.com/scripts/ (by officialjackofalltrades)
- **核心逻辑**: 将摆动点分类与实时买卖量Delta分析融合。每个确认的摆动高低点不仅按价格测量，还按产生该摆动的净成交量组成进行测量——揭示结构性移动是由真正的机构买/卖推动，还是低信心、低成交量的试探。
- **技术指标**: Swing Detection(非重绘)，Volume Delta(每条腿累积)，BOS/ChoCH检测，Structure Cloud
- **策略类型**: 市场结构 + 成交量分析 / 机构行为识别
- **适用市场/时间框架**: 通用，Swing Length可调(5-200)
- **创新点**:
  - 按结构性腿(leg)计算成交量Delta，而非按K线或全局指标
  - Zone颜色编码反映成交量Delta梯度：深绿=高买入Delta(机构买入)，深红=高卖出Delta(机构卖出)
  - 摆动确认使用lookback比较模式，bar收盘确认，永不重绘
  - BOS和ChoCH检测带成交量Delta上下文，比单独任一信号更完整

## 5. Candle Volume Architecture [JOAT]

- **策略名称**: Candle Volume Architecture
- **URL**: tradingview.com/scripts/ (by officialjackofalltrades)
- **核心逻辑**: 为每个检测到的摆动构建基于价格的成交量分布profile，识别控制点(POC)，计算可配置的价值区域(Value Area, 默认70%)，直接在价格图上渲染可视化的"成交量架构"。
- **技术指标**: Swing Detection, Volume Profile (POC, Value Area), ATR
- **策略类型**: 成交量剖面 / 价格结构分析
- **适用市场/时间框架**: 通用，极快时间框架(1分钟以下)可能效果不佳
- **创新点**:
  - 将Volume Profile方法应用于自动检测的价格摆动而非固定日历周期
  - 不通明度缩放的bin条形图：最活跃水平立即显而易见
  - POC接近检测：当价格在活跃POC的0.3 ATR范围内时仪表板高亮显示
  - Profile形状反映情绪：POC接近高点=看涨分布，POC接近低点=看跌分布

## 6. Momentum Terrain 3D [LuxAlgo]

- **策略名称**: Momentum Terrain 3D
- **URL**: tradingview.com/scripts/ (by LuxAlgo)
- **核心逻辑**: 将多个RSI回溯长度映射到时间和RSI值维度，创建3D地形景观。X轴=时间，Y轴=RSI回溯长度(前端=快，后端=慢)，Z轴=RSI值。提供多维度动量视图。
- **技术指标**: 多长度RSI矩阵，3D渲染引擎(Pine Script polyline + matrix)
- **策略类型**: 多维度动量可视化
- **适用市场/时间框架**: 通用
- **创新点**:
  - 3D地形可视化多时间框架RSI动量
  - 统一动量：整个地形从前到后一起上升=所有时间框架看涨一致
  - 趋势压缩：地形平坦停留在50中性平面附近=盘整或犹豫
  - OB/OS柱：红色柱(RSI>70)和青色柱(RSI<30)标记极端动量
  - 使用三角变换(偏航和俯仰)将矩阵投影到3D空间

## 7. ORB Retest Entry

- **策略名称**: ORB Retest Entry with Time-Limited Logic
- **URL**: tradingview.com/scripts/ (by Trend_MyFrend)
- **核心逻辑**: 高级开盘区间突破(ORB)指标，带有时间限制有效性窗口。自动检测开盘区间，等待突破，然后仅在价格在用户定义的时间窗口内回测原始水平时触发信号。
- **技术指标**: ORB (Opening Range Breakout)，时间限制窗口
- **策略类型**: 日内交易 / 开盘区间突破
- **适用市场/时间框架**: 日内，可自定义ORB时间框架(默认15分钟)
- **创新点**:
  - 四阶段流程：区间识别→突破检测→回测确认→风险管理
  - 时间限制有效性：信号仅在指定时间窗口内有效(如10:15前)
  - 半区间止损：SL自动设为ORB区间的50%
  - 智能线条管理：入场/SL/TP线实时延伸但在被触发后立即冻结
  - 蓝色(区间构建中)→绿/红(信号生成)的盒子颜色反馈系统

## 14. Market Structure Adaptive RSI

- **策略名称**: Market Structure Adaptive RSI
- **URL**: tradingview.com/scripts/ (by Uncle_the_shooter)
- **核心逻辑**: 根据市场结构事件(CHOCH和BOS)调整RSI平滑。CHOCH后切换到正常模式(无平滑/回撤模式)，BOS后切换到趋势模式(平滑)。在趋势中随着连续BOS/CHOCH增加，逐步减少RSI平滑长度使其更快反应。
- **技术指标**: RSI(自适应平滑)，多Pivot市场结构检测，CHOCH/BOS
- **策略类型**: 自适应动量 / 市场结构
- **适用市场/时间框架**: 通用
- **创新点**:
  - RSI制度切换：趋势模式(平滑)vs回撤模式(正常)
  - 自适应平滑：连续结构性事件越多→平滑长度越短→RSI越快
  - 多Pivot市场结构分析：同时使用5个不同pivot长度
  - 事件优先级：多突破蜡烛上CHOCH优先于BOS
  - 在延伸趋势中更容易发现早期回撤迹象
